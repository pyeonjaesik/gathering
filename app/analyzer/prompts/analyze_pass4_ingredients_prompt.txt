아래 원재료 텍스트를 구조화하세요. (PASS4-INGREDIENTS)

입력:
- 원재료 텍스트: __INGREDIENTS_TEXT__

목표:
1) 원재료 배열 구조화

규칙:
- JSON 외 텍스트 출력 금지
- 확실하지 않으면 추정하지 말고 null 사용
- 원문(입력 원재료 텍스트)에 없는 단어/국가/원산지 정보를 추가하지 말 것
- 특히 origin_detail에 국가명을 임의로 보정/추정/자동완성하지 말 것
- 원문 근거가 없으면 origin/origin_detail은 null
- 원재료 항목 순서는 원문 순서를 최대한 유지
- "우유, 대두 함유" 등 알레르기 안내는 원재료 항목으로 넣지 말고 ignored_fragments에 넣기
- 원재료명이 부자연스럽거나 잘렸다고 판단되면 ingredient_name=null
- 괄호/대괄호/중괄호 안 텍스트는 기본적으로 하위정보로 취급
- 애매하면 승격 금지:
  - 내부 텍스트를 ingredient_name으로 올리지 말고 sub_ingredients로 유지
  - 상위 항목명이 불명확하면 ingredient_name=null 처리
- 원산지는 origin(상위 분류) / origin_detail(상세)로 분리

용어 정의(중요):
- "승격" = 괄호/대괄호/중괄호 내부의 하위 설명 텍스트를 메인 ingredient_name으로 올리는 것
- 이 작업에서는 승격을 금지합니다.

원산지 귀속 규칙(중요):
- "X: 미국산, 호주산" 형태는 X의 하위 원재료가 아니라 X의 원산지 상세 나열입니다.
- 따라서 이 경우:
  - X는 name(또는 ingredient_name)
  - origin_detail="미국산, 호주산"
  - name=null인 하위 노드를 만들지 마세요.
- 이름 없는 노드(name=null)는 생성하지 마세요.
- 콜론(:) 뒤가 전부 원산지/원산지 나열이면, 이를 하위항목으로 분해하지 말고 동일 노드의 origin/origin_detail로 귀속하세요.
- 특히 "밀:미국산, 호주산"은 아래처럼 해석합니다:
  - name="밀"
  - origin=null
  - origin_detail="미국산, 호주산"
  - sub_ingredients=[]
- 아래 해석은 금지:
  - name="밀"
  - sub_ingredients=[{name:null, origin:"미국산"}, {name:null, origin:"호주산"}]

예시:
- 입력: 밀가루(밀:미국산, 호주산)
- 올바른 해석:
  - ingredient_name=밀가루
  - sub_ingredients[0].name=밀
  - sub_ingredients[0].origin=null
  - sub_ingredients[0].origin_detail=미국산, 호주산
  - sub_ingredients[0].sub_ingredients=[]
- 잘못된 해석(금지):
  - sub_ingredients[0].name=밀
    - sub_ingredients=[{name:null, origin:미국산}, {name:null, origin:호주산}]

      "amount": "문자열 또는 null",
      "sub_ingredients": [
        {
          "name": "문자열 또는 null",
          "origin": "문자열 또는 null",
          "origin_detail": "문자열 또는 null",
          "amount": "문자열 또는 null",

        }
      ]
    }
  ],
  "ignored_fragments": ["문자열", "..."],
  "reason": "간단한 근거"
}
