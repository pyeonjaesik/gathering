이 이미지를 체크리스트 기준으로만 평가하세요. (PASS2-B: 정보 존재 판정)

파이프라인 맥락(중요):
- PASS2-B는 PASS2-A를 통과한 이미지를 대상으로 수행됩니다.
- 즉, 이 이미지는 "품질적으로는 어느 정도 읽을 수 있다"고 가정하고,
  이제 실제로 필요한 정보가 "명확히 존재하는지"만 판정합니다.
- PASS2-B 결과는 다음 추출 단계(PASS3) 진입 조건으로 사용됩니다.

최종 서비스 목표:
- 원재료명, 품목보고번호, 제품명, 영양성분을 신뢰도 높게 수집.
- 오탐(없는 정보를 있다고 판단) 비용이 크므로 보수적으로 판정하세요.

중요 규칙:
- 아래 항목만 true/false로 반환하세요.
- 값을 모르면 보수적으로 false를 사용하세요.
- JSON 객체만 출력하고, 설명 문장/코드블록은 출력하지 마세요.
- "대충 보인다", "추정 가능", "비슷해 보인다"는 false입니다.
- 특히 품목보고번호는 라벨만으로 true 금지:
  - 라벨 + 실제 숫자열을 함께 읽을 수 있을 때만 true
- 복수 후보(복수 제품명/복수 원재료표)가 보이면 관련 항목은 false

이 단계가 보는 이미지 유형:
- 일반적으로 식품 라벨/성분표 이미지(정면/평면 또는 충분히 읽히는 상태)
- 단, PASS2-A를 통과했더라도 정보가 실제로 없거나 불완전하면 false

항목 정의 (모든 항목은 true가 "적합/좋은 상태" 의미):
- has_ingredients_section:
  - true: "원재료명", "원재료", "INGREDIENTS" 라벨 또는 목록이 명확히 읽힘
  - false: 라벨/목록 확인 불가, 일부만 보여 완전하지 않음
- has_report_number_label:
  - true: "품목보고번호" 또는 "품목제조보고번호" 라벨 + 실제 번호 숫자열이 함께 명확히 읽힘
  - false: 라벨만 보임, 숫자열 불명확, 숫자 일부만 보임
- has_product_name:
  - true: 제품명이 완전하게 식별 가능
  - false: 식별 불가/부분 누락/복수 후보
- has_nutrition_section:
  - true: "영양정보", "영양성분", "Nutrition Facts" 표/영역 확인 가능
  - false: 확인 불가
- reason:
  - 핵심 근거 1문장 (왜 true/false인지)

응답 JSON 스키마:
{
  "has_ingredients_section": true,
  "has_report_number_label": true,
  "has_product_name": true,
  "has_nutrition_section": true,
  "reason": "원재료/품목보고번호/제품명이 명확히 확인됨"
}
