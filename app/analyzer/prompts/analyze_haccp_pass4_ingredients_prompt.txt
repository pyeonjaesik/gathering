아래 HACCP 원재료 텍스트를 구조화하세요. (HACCP-PASS4-ING, PROMPT-FIRST)

입력:
- 원재료 텍스트: __RAWMTRL_TEXT__

목표:
- HACCP 원문 그대로를 기준으로 `ingredients_items`를 구조화
- 코드 보정에 의존하지 않도록, 응답에서 분류를 정확히 완료

절대 규칙:
- JSON 외 텍스트 출력 금지
- 원문에 없는 단어/원산지/함량/노드 추가 금지
- 확실하지 않으면 null
- 원문 순서 유지
- `name:null`, `ingredient_name:null` 노드 생성 금지
- 빈 문자열 name/ingredient_name 금지

구조 규칙:
- 최상위는 반드시 `ingredient_name` 사용
- 하위는 `name` 사용
- 괄호/대괄호/중괄호 내부는 기본적으로 해당 부모의 `sub_ingredients`
- `[(...)]`, `{[...]}` 같이 중첩 기호가 섞여도, 닫히는 기호까지는 부모 내부로 유지
- 브라켓이 닫힌 뒤 이어지는 텍스트는 부모 바깥(형제 항목)으로 복원

의미 규칙(중요):
- `국산`, `수입산`, `외국산`, `외국산:미국, 인도`, `미국산/호주산` 등은
  - 하위 원재료가 아니라 `origin`에 귀속
  - 원재료명(name/ingredient_name)으로 만들지 말 것
- 원산지의 상세표현(예: `미국산, 호주산`, `외국산:미국, 인도`)도 전부 `origin`에 기록
- `N종`, `%`, `g`, `mg`, `ml`, `고형분 xx%` 같은 수량/비율은 `amount`
- `합성보존료`, `향미증진제`, `산도조절제` 등 기능/용도 표현은:
  - 원문이 괄호 안 설명이어도 `origin`으로 보내지 말 것
  - 기본적으로 해당 원재료의 `sub_ingredients[].name`으로 분류
  - 예: `파라옥시안식향산에틸(합성보존료)` -> `파라옥시안식향산에틸`의 하위 원재료 `합성보존료`
- `원재료명 N종` 패턴은 `ingredient_name/name=원재료명`, `amount=N종`
- `A-B` 또는 `A:B` 패턴은 분리 가능한 경우만 분리:
  - 뒤쪽이 원산지/수량 정보면 `origin`/`amount`
  - 뒤쪽이 실제 원재료명이면 하위 원재료로 분리
  - 모호하면 원문 유지 후 `ignored_fragments` 최소 사용

정합성 체크(응답 전 자체 점검):
1) 모든 노드에 name 또는 ingredient_name 존재 여부 확인
2) `origin`에 `합성보존료`, `향미증진제`, `산도조절제` 같은 용도어가 들어가면 반드시 수정
3) `amount`에 `국산`, `외국산` 같은 원산지 표기가 들어갔는지 확인
4) `국산/수입산/외국산`이 하위 노드로 잘못 생성됐는지 확인
5) 하위 노드가 없는데 괄호/브라켓 내부 텍스트가 누락되지 않았는지 확인

응답 JSON 스키마:
{
  "ingredients_items": [
    {
      "ingredient_name": "문자열",
      "origin": "문자열 또는 null",
      "amount": "문자열 또는 null",
      "sub_ingredients": [
        {
          "name": "문자열",
          "origin": "문자열 또는 null",
          "amount": "문자열 또는 null",
          "sub_ingredients": [ "동일 스키마 재귀" ]
        }
      ]
    }
  ],
  "ignored_fragments": ["문자열", "..."]
}
