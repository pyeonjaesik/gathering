아래 HACCP 영양성분 텍스트를 구조화하세요. (HACCP-PASS4-NUT)

입력:
- 영양성분 텍스트: __NUTRIENT_TEXT__

목표:
- nutrition_items를 원문 기준으로 구조화
- 기준량(몇 g/ml 당)을 nutrition_basis로 구조화

규칙:
- JSON 외 텍스트 출력 금지
- 원문에 없는 영양성분명/값/단위를 추가하지 말 것
- 값/단위가 불명확하면 null
- 일일기준치(%)가 보이면 daily_value에 기록, 없으면 null
- 원문 순서를 최대한 유지
- 기준량 추출 규칙:
  - 1회를 나타내는 뜻을 의미하는 텍스트가 있을 때, 이를 기준량으로 우선 기록.
    예시.1회 제공량, 1회 함량, 1봉지, 1포, 1캔 등을 기준량으로 우선 기록.
  - g/ml 없이 "1개", "1포", "1봉지", "1알" 등만 있으면 해당 개수 단위를 기준량 단위로 기록
  - "N개(...g/ml)"처럼 개수와 g/ml가 함께 있으면 더 작은 단위인 g/ml를 기준량으로 기록
  - "1개당"과 "총내용량당"이 함께 있으면 더 작은 단위인 "1개당"을 기준량으로 기록
  - 작은 단위를 기준량으로 선택했으면, nutrition_items도 반드시 해당 단위 섹션의 값만 사용
    예: "<1개당 ...>" 섹션을 선택했으면 "<총 내용량당 ...>" 값은 섞지 말 것
  - "총내용량 500g 100g당 400kcal"처럼 총내용량과 "N g/ml당"이 함께 있으면 "N g/ml당"을 기준량으로 기록
  - 1회 기준이 없고 "N g/ml당"도 없고 "총내용량"만 있으면 총내용량을 기준량으로 기록
  - 위 조건들이 없을 때 "100g당", "100ml당", "N g/ml당" 표현을 기준량으로 기록
  - 없으면 nutrition_basis는 null

응답 JSON 스키마:
{
  "nutrition_basis": {
    "per_amount": "기준량 수치 또는 null",
    "per_unit": "g/ml 또는 null",
    "basis_text": "원문 기준량 문구 또는 null"
  },
  "nutrition_items": [
    {
      "name": "영양성분명 또는 null",
      "value": "값 또는 null",
      "unit": "단위 또는 null",
      "daily_value": "일일기준치(%) 또는 null"
    }
  ]
}
