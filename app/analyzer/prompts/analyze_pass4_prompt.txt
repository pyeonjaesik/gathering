아래 식품 라벨 텍스트를 구조화하세요.

입력:
- 원재료 텍스트: __INGREDIENTS_TEXT__
- 영양성분 텍스트: __NUTRITION_TEXT__

목표:
1) 원재료 배열 구조화
2) (있다면) 영양성분 JSON 파싱

규칙:
- JSON 외 텍스트 출력 금지
- 확실하지 않으면 추정하지 말고 null 사용
- 원재료 항목 순서는 원문 순서를 최대한 유지
- "우유, 대두 함유" 등 알레르기 안내는 원재료 항목으로 넣지 말고 ignored_fragments에 넣기
- 원재료명이 부자연스럽거나 잘렸다고 판단되면 ingredient_name=null

응답 JSON 스키마:
{
  "ingredients_items": [
    {
      "ingredient_name": "문자열 또는 null",
      "origin": "문자열 또는 null",
      "amount": "문자열 또는 null"
    }
  ],
  "nutrition_items": [
    {
      "name": "영양성분명 또는 null",
      "value": "수치/문자열 또는 null",
      "unit": "단위 또는 null",
      "daily_value": "일일기준치 % 또는 null"
    }
  ],
  "ignored_fragments": ["문자열", "..."],
  "reason": "간단한 근거"
}
